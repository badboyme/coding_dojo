My essays

My last post depends on each other, and should be read in sequence. Articles not in bold are optional.

  • Assignment Statement Considered Harmful. Pervasive mutable state make programs more difficult to deal with. While this is old news. most programmers don't know or don't care. It's a pitty, simple remedies exist.
  • Defining Syntactic Sugar. I sometimes hear arguments about what is or is not syntactic sugar. This is an attempt at defining it.
  • Class-based Programming as Sugar. Most of the time, "Object Oriented" actually mean classes. Classes are nothing more than syntactic sugar, at least in statically typed languages such as Java and C++. Sugar is not useless, but seeing past it heps us understand classes better.
  • Taboo "OO" is so overloaded the we should stop using it. Better substitutes are "Classes" and "Prototypes".
  • How Class based Programing Suck. classes are vastly sub-optimal. Functional programming is far better.
